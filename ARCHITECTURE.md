# Архитектура Проекта (SkaldEngine Frontend)

Этот документ предназначен для разработчиков, которые хотят разобраться в устройстве проекта и его структуре.

## 🏗 Архитектурный подход
Проект следует **Feature-Based** архитектуре. Это означает, что код разделен по функциональным областям, а не по техническим слоям (компоненты/хуки/типы). Это упрощает масштабирование и навигацию.

## 📂 Структура папок

### `src/core/`
Core-слой содержит общую логику, которая не привязана к конкретной фиче:
- `api/`: Централизованный `ApiClient` (на базе axios) и сервисы для работы с бэкендом.
- `types/`: Глобальные интерфейсы и типы, согласованные с бэкенд-схемами.
- `hooks/`: Общие React-хуки (например, для работы с Dashboard).

### `src/features/`
Каждая папка здесь — это изолированная зона функциональности:
- `Auth/`: Логика входа, регистрации и защиты маршрутов.
- `Chat/`: Основной игровой экран чата, стриминг сообщений.
- `Dashboard/`: Главная консоль выбора персонажей.
- `CreateChat/`: Модальное окно настройки нового чата.
- `Landing/`: Посадочная страница для гостей.

Внутри каждой фичи могут быть свои папки `components/`, `api/`, `store/` — это помогает держать связанный код рядом.

### `src/theme/`
Единый источник правды для дизайна:
- `global/`: Основные стили и `tokens.css` (дизайн-токены: цвета, тени, отступы).
- `screens/`: Стили, специфичные для конкретных экранов.
- `components/`: Стили для общих UI-элементов.

### `src/components/ui/`
Библиотека примитивных переиспользуемых компонентов (кнопки, индикаторы загрузки, навбар, подвал).

## 🚀 Ключевые принципы разработки

### 1. Типизация (TypeScript)
Мы используем строгую типизацию. Избегайте использования `any`. Все интерфейсы данных должны соответствовать Pydantic-схемам бэкенда.

### 2. Path Aliases
Для импортов используйте алиас `@/`, который указывает на папку `src/`.
*Плохо:* `../../core/api/client`
*Хорошо:* `@/core/api/client`

### 3. API Client
Все запросы к бэкенду должны идти через `ApiClient` в `src/core/api/client.ts`. Он автоматически обрабатывает авторизацию через localStorage и унифицирует ошибки.

### 4. Дизайн-система
Используйте CSS-переменные из `src/theme/global/tokens.css` вместо "хардкода" цветов. Это обеспечит визуальную целостность проекта.
